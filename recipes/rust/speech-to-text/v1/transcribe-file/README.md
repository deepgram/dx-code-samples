# Transcribe File

Transcribes audio from a local file buffer using Deepgram's pre-recorded speech-to-text API. This demonstrates how to work with audio data loaded into memory rather than streaming from a URL.

## What this feature does

File-based transcription allows you to process local audio files or audio data already loaded into memory. This is useful when you have audio files stored locally, received via upload, or generated by your application.

## Key parameters

- **AudioSource::from_buffer_with_mime_type()**: Creates audio source from in-memory buffer with content type
- **MIME type**: Specifies audio format (audio/wav, audio/mp3, etc.) for proper processing

## Sample output

When you run this example with the demo audio, you'll see output like:

```
NASA that's one small step for man one giant leap for mankind
```

## Prerequisites

- Set `DEEPGRAM_API_KEY` environment variable
- Internet connection to download the demo audio file

## Run the example

```bash
cargo run
```
